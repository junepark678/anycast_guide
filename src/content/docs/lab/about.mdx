# Anycast Lab

Anycast Lab is a local-first network workbench for experimenting alongside the
guide. Build routers, clients, services, and links; edit the same BIRD and FRR
configuration files you would deploy; vary delay and packet loss; fail links;
and inspect routing decisions with packet traces.

<LabFollowAlong />

<Aside type="note" title="Runtime fidelity">
The lab always identifies the selected runtime. **SIM** is the fast,
deterministic compatibility engine. **NATIVE VM** runs real i686 BIRD and FRR
binaries inside an isolated Linux guest powered by the browser's v86/WebAssembly
emulator. Native mode never silently falls back to SIM. Native files are
preserved unchanged in either mode.
</Aside>

<Aside type="tip" title="FRR protocol daemons">
If an FRR project omits `/etc/frr/daemons`, native mode supplies a conservative
BGP-only activation file. Add your normal `/etc/frr/daemons` to enable the
packaged OSPF, IS-IS, BFD, RIP, Babel, or PIM daemons. The lab passes a supplied
file through unchanged. While native VMs are running or paused, configuration
and topology edits are locked so the saved project cannot drift away from what
the daemons actually tested; reset the runtime before editing.
</Aside>

## Local data

Projects are stored in this browser using IndexedDB. Browser storage can be
cleared by the user or operating system, so export important projects
periodically. Configuration and topology data are never sent to anycast.guide.

The lab is open source in the
[anycast_lab repository](https://github.com/junepark678/anycast_lab).
